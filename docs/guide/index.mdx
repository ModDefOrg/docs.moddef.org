# Guide

Hand-written tutorials and how-tos for working with ModDef. Start with
[Getting started](./getting-started.mdx), then dig into the concept pages as you
need them.

- **[Getting started](./getting-started.mdx)** — read your first device in each language.
- **[What is a ModDef document?](./concepts/document-model.mdx)** — the file, its three encodings, and how it is validated.
- **[Types & storage](./concepts/types-and-storage.mdx)** — storage widths, byte/word order, primitives.
- **[Transforms & scaling](./concepts/transforms.mdx)** — static rationals, SunSpec scale factors, selector cases.
- **[Measurands](./concepts/measurands.mdx)** — the semantic layer that lets you query by meaning, not address.
- **[Writing & constraints](./concepts/writes.mdx)** — access modes and §11.4 validation.
- **[SunSpec discovery](./concepts/discovery.mdx)** — dynamic, model-chained addressing.

For the normative rules, see the **[Specification](/spec/)**.
